# NutriMind 前端

NutriMind 是面向运动营养管理场景的 Vue 3 单页应用。项目已完成可运行的最小版本，并按普通用户和管理员两种身份提供彼此独立的工作区。

- 普通用户端：AI 营养教练、食物照片识别、运动营养库、个人资料与身体目标。
- 管理员端：系统总览、用户管理、目标检测任务监控、模型训练管理。
- 默认前端地址：`http://localhost:3000`
- 默认后端地址：`http://localhost:9999`

## 技术栈

- Vue 3、Composition API、`<script setup>`
- Vite 7、Vue Router 4、Pinia 3
- Axios、Element Plus、Phosphor Icons Vue
- Sass、Vitest

## 快速启动

### 1. 前置条件

- 安装 Node.js 20 或更高版本。
- 后端服务已经启动，并且数据库、模型与大模型相关配置由后端项目正确提供。

### 2. 安装并启动前端

```powershell
cd frontend
npm install
npm run dev
```

浏览器访问：

```text
http://localhost:3000
```

开发服务器会把 `/api` 请求代理到 `http://localhost:9999`，因此通常不需要在浏览器中额外处理跨域。

### 3. 修改后端地址

后端不在默认地址时，复制环境变量示例：

```powershell
Copy-Item .env.example .env
```

然后修改 `.env`：

```env
VITE_API_TARGET=http://localhost:9999
```

修改后需要重新启动前端开发服务器。

## 手机端访问

确保手机和电脑连接同一局域网，然后使用以下命令让开发服务器监听局域网地址：

```powershell
npm run dev -- --host 0.0.0.0
```

查询电脑的局域网 IPv4 地址后，在手机浏览器中访问：

```text
http://电脑IPv4地址:3000
```

例如电脑地址是 `192.168.1.20`，手机访问 `http://192.168.1.20:3000`。如果无法打开，请检查 Windows 防火墙是否允许 Node.js 或 TCP 3000 端口通过专用网络。

## 身份与页面入口

### 普通用户

注册或使用普通账号登录后，默认进入：

```text
/app/coach
```

| 页面 | 地址 | 主要功能 |
|---|---|---|
| AI 营养教练 | `/app/coach` | 文本/图片对话、会话历史、删除会话、停止等待回复 |
| 食物照片 | `/app/scan` | 图片检测、识别结果、照片保存、历史查看与删除 |
| 运动营养库 | `/app/library` | 文档上传、问答、语义检索、来源删除、统计 |
| 身体与目标 | `/app/profile` | 头像、个人资料、身体基准、营养与训练目标 |

### 管理员

管理员登录后会自动进入：

```text
/admin/dashboard
```

| 页面 | 地址 | 主要功能 |
|---|---|---|
| 系统总览 | `/admin/dashboard` | 用户、检测、训练和食物数据概览 |
| 用户管理 | `/admin/users` | 用户检索、状态、角色与管理员身份管理 |
| 检测监控 | `/admin/detections` | 检测统计、状态分布、任务列表与详情 |
| 模型训练 | `/admin/training` | 训练任务、指标、启停操作和模型文件管理 |

当前登录页不再提供开发阶段的页面预览入口。进入管理员页面必须使用真实管理员账号，后端用户记录需要满足：

```text
is_superuser = true
```

如果已有账号需要被设置为管理员，应由有数据库权限的人员在 PostgreSQL 中执行一次：

```sql
UPDATE users
SET is_superuser = TRUE
WHERE username = '你的用户名';
```

修改后需要退出并重新登录。普通用户直接访问 `/admin/*` 会被前端送回 `/app/coach`；后端管理员接口也会再次校验权限。仅在角色列表中添加 `admin` 可能让前端识别身份，但不满足当前后端对 `is_superuser` 的严格检查。

## 已接入功能

### 认证与账户

- 注册、登录、退出、会话恢复和修改密码。
- JWT 由后端通过 HttpOnly Cookie 提供，Axios 请求携带 `withCredentials`。
- 登录后请求当前用户信息，并根据身份进入对应工作区。
- 遇到失效登录状态时清理前端会话并返回登录页。

### AI 营养教练

- 创建、查询、切换和删除后端持久化会话。
- 发送文本消息或携带图片提问。
- 展示 Markdown 回复与智能体工具调用信息。
- 点击“停止生成”可通过 `AbortController` 立即中止浏览器当前请求和等待状态。
- 教练页右侧计划卡片读取真实个人资料，不再展示固定热量、蛋白质或训练数据。

### 食物照片与目标检测

- 选择或拍摄图片后调用目标检测接口。
- 展示识别类别、置信度、目标框和处理结果。
- 检测与照片保存是两个独立操作，页面不会把“保存成功”描述成“检测成功”。
- 支持上传历史、原图查看和删除。

### 运动营养库

- 上传 PDF、Markdown、TXT/TEXT 和后端支持的图片资料。
- 直接问答或查看语义检索结果。
- 展示知识库统计并按来源删除资料。
- 智能体是否在某次对话中检索知识库，由后端工具选择决定。

### 个人资料与身体目标

- 获取并更新后端保存的账户、身体基准和目标资料。
- 上传、替换和删除头像；更新后同步刷新全局导航头像。
- 身体目标表单拆分为“主要目标、身体基准、营养训练”三个步骤。
- 教练页面可以读取这些数据用于界面展示；是否自动注入 AI 上下文由后端负责，前端不会把完整个人资料附加到每条消息。

### 管理员能力

- Dashboard 聚合统计。
- 用户分页、详情、启停、角色和管理员身份维护。
- 目标检测统计、任务分页和任务详情。
- 模型训练任务创建、启动、暂停、指标查看与模型文件删除。

## 当前边界

以下内容尚未形成完整的前后端能力，不能作为已经交付的功能描述：

- “停止生成”目前只保证前端停止等待；后端没有提供 `request_id` 与取消接口时，不能保证模型服务端已经真正停止推理。
- 前端尚未提供 Thinking 开关和模型切换，因为当前接口没有可用模型列表、能力声明及对应请求字段。
- 用户对话的长期记忆与建议沉淀尚未接入；当前营养图谱仅展示食物、分类和营养素关系。
- AI 是否自动读取身体资料作为回答上下文属于后端智能体上下文能力；后端保持现有聊天契约时，无需修改前端。
- 食物识别效果、训练任务执行和 AI 回复质量依赖后端模型、数据集、第三方模型服务及运行环境。

## 开发约定

- 新增请求统一放在 `src/api/`，不要在页面中直接拼接接口地址。
- 不稳定的接口响应先在 `src/utils/` 中适配，页面只使用统一的数据结构。
- 缺失值与真实数值 `0` 必须区分。
- 不在前端伪造热量、目标、训练、检测或系统统计数据。
- 前端路由守卫只负责用户体验，资源归属与管理员权限必须由后端校验。
- 普通用户数据由后端根据当前 Cookie/JWT 身份过滤，不能信任前端传入任意 `user_id`。
- 延续深色石墨与能量绿的视觉体系，并保证键盘焦点、错误恢复、移动端布局和约 44px 的最小触控区域。

- 后端 Swagger：`http://localhost:9999/docs`
