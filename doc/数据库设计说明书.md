# NutriMind-Agent 数据库设计说明书

> 版本：0.1.0  
> 数据库：PostgreSQL 15+（含 pgvector 扩展）  
> ORM：SQLAlchemy 2.0  
> 日期：2026-07-16

---

## 一、概述

NutriMind-Agent 是基于 YOLOv11 的智能营养分析平台。数据库采用 PostgreSQL，并通过 pgvector 扩展支持向量存储，用于知识库语义检索。

### 1.1 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| 数据库 | PostgreSQL 15+ | 主存储 |
| 向量扩展 | pgvector | 知识库 Embedding 存储 |
| ORM | SQLAlchemy 2.0 | Python 对象映射 |
| 密码哈希 | bcrypt | 用户密码加密 |
| Token | JWT (HS256) | 无状态认证 |

### 1.2 表清单

| # | 表名 | 说明 | 模块 |
|---|------|------|------|
| 1 | `users` | 用户账户 | 认证 |
| 2 | `roles` | 角色定义 | 权限 |
| 3 | `permissions` | 权限定义 | 权限 |
| 4 | `user_roles` | 用户-角色关联 | 权限 |
| 5 | `role_permissions` | 角色-权限关联 | 权限 |
| 6 | `body_profiles` | 用户身体资料 | 个人资料 |
| 7 | `goal_profiles` | 用户目标配置 | 个人资料 |
| 8 | `detection_scenes` | 检测场景配置 | 目标检测 |
| 9 | `detection_tasks` | 检测任务记录 | 目标检测 |
| 10 | `training_tasks` | 训练任务记录 | 模型训练 |
| 11 | `food_nutrition` | 食物营养数据 | 知识库 |

> 另有 `langchain_pg_embedding` 和 `langchain_pg_collection` 两张表由 LangChain 自动管理，用于知识库向量存储，不在本文档详述。

## 三、表详细设计

### 3.1 users — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO | 用户 ID |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL, INDEX | 用户名 |
| `email` | VARCHAR(100) | UNIQUE, NOT NULL, INDEX | 邮箱 |
| `hashed_password` | VARCHAR(255) | NOT NULL | bcrypt 加密密码 |

## 四、枚举值说明

| 值 | 含义 |
|------|------|

| 值 | 含义 |

| `completed` | 训练完成 |
| `failed` | 训练失败 |
| `paused` | 已暂停 |

### 4.3 目标模式 `mode`

| 值 | 含义 |
|------|------|
| `cut` | 减脂 |
| `muscle` | 增肌 |
| `maintain` | 保持体态 |

### 4.4 活动水平 `activity_level`

| 值 | 含义 |
|------|------|
| `sedentary` | 久坐 |
| `light` | 轻度活动 |
| `moderate` | 中等活动 |
| `high` | 高活动量 |
| `very_high` | 极高活动量 |

## 五、数据安全与隔离

| 模块 | 隔离策略 |
|------|------|
| 认证 (users) | 密码 bcrypt 哈希，JWT 无状态认证 |
| 知识库 | 上传时在 metadata 中存储 `user_id`，搜索/删除/统计按用户过滤 |
| 摄像头 (camera) | 文件名内置 `user_id` 前缀，列表/查看/删除均校验所有权 |
| 个人资料 (body/goal) | `user_id` UNIQUE 一对一，仅当前用户可读写 |
| 看板 (dashboard) | 仅 `is_superuser=TRUE` 可访问 |

## 七、数据库部署

```sql
-- 创建 pgvector 扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 确认扩展
SELECT * FROM pg_extension WHERE extname = 'vector';
```

项目启动时通过 SQLAlchemy `Base.metadata.create_all()` 自动建表。
